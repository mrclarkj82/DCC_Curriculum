# Media Submission Guidelines

## Acceptable Submission Types

- Video file upload placeholder
- Video link placeholder
- File upload placeholder
- Text reflection
- Production notes
- Thumbnail or still image when requested

## Video File Upload Placeholder

Future app phases may allow students to upload video evidence through Cloud Storage for Firebase. Do not build uploads in this phase.

## Video Link Placeholder

Students may eventually submit hosted links when approved by the teacher. Links must be viewable by the teacher and should not expose private student media publicly.

## File Upload Placeholder

Future file uploads may include scripts, shot lists, thumbnails, images, or production notes depending on the project.

## Reflection Requirement

Every media submission should include a short reflection explaining the student role, contribution, technical skill, creative decision, and next improvement.

## Teacher Review Status Definitions

- Draft: student has not submitted final evidence.
- Submitted: evidence is ready for teacher review.
- Reviewed: teacher has checked the work and may leave feedback.
- Returned: teacher has requested revision or missing evidence.

## Revision Workflow Concept

Students submit evidence, teachers review it, and returned work should clearly identify what needs revision. Future app phases should preserve submission history and teacher feedback.

## Privacy Reminder

Do not commit real student data, private student media, private links, rosters, credentials, or unpublished school media to this repository.
