## Big Idea

Video editing is not one magic button. It is a sequence of organized post-production passes: media, edit, audio, graphics, color, and delivery.

## Classroom Hook

Show a messy folder of random clips next to a clean project timeline. Ask which one would be easier to finish under deadline and why.

## Student-Friendly Analogy

Editing is like building a school broadcast segment: collect the ingredients, sort them, assemble the story, polish sound and image, then deliver a version others can watch.

## Why This Matters For Digital Content Creators

Students who can manage footage, make clean edits, and export correctly can contribute to broadcasts, social media packages, interviews, event recaps, and later group projects.

## Misconception Warnings

- Students may think Resolve projects are the same thing as exported video files.
- Students may skip naming conventions because no timeline exists yet.
- Students may assume every page is for editing instead of a specialized post-production task.

## Teacher Demo Sequence

1. Open DaVinci Resolve and locate the Project Manager.
2. Create a new project using the class naming convention.
3. Identify the Media, Cut, Edit, Fusion, Color, Fairlight, and Deliver pages.
4. Write a one-sentence job description for at least four Resolve pages.
5. Export or screenshot proof of the project setup according to teacher directions.
6. Save and close the project cleanly.

## Suggested 15-20 Slide Structure

1. Title and learning target: establish the workflow and evidence goal.
2. Bell ringer: What is one problem that can happen if an editor starts cutting video before naming, saving, and organizing the project?
3. Big idea: Resolve work happens in passes, not all at once.
4. Why creators care: connect the workflow to broadcast, YouTube, interviews, or client-style work.
5. Transcript anchor: use 00:00:00-00:13:16 from the stored tutorial source.
6. Key vocabulary: DaVinci Resolve, Project Manager, Project Library, Resolve page, Post-production, Workspace.
7. Interface map: show the Resolve page, panel, or tool students need today.
8. Teacher demo part 1: model the first half of the workflow slowly.
9. Teacher demo part 2: model the evidence or quality check.
10. Common mistakes: show one broken example and one fixed example.
11. Guided checkpoint: students point to the correct panel/tool before independent work.
12. Assignment launch: Resolve Project Setup Proof.
13. Evidence checklist: show exactly what link-based evidence is required.
14. Work time checkpoint 1: teacher checks setup and naming.
15. Work time checkpoint 2: teacher checks the visible Resolve result.
16. Extension challenge: Create a one-page Resolve workflow map showing which page you would use for media, editing, color, audio, and export.
17. Studio challenge: Design a naming convention for a class editing project and explain how it prevents lost work.
18. Reflection and exit ticket: Which Resolve page or startup area did you use today, and what job does it help an editor do?

## Accessibility Guidance

- Use high contrast text over dark synthwave backgrounds.
- Keep slide text short and leave detailed instructions in speaker notes.
- Add alt text for every screenshot, diagram, and generated image.
- Avoid relying on color alone to show correct and incorrect examples.
- Make all interface labels large enough to read from the back of the room.
- Do not use flashing animations or fast strobe effects.
