# Presentation Brief: Video Game Development File Organization

This premium brief is for creating a 15-20 slide teacher-led PowerPoint for the first Q2 game development organization lesson.

## Audience

High school Digital Content Creators students who have completed Q1 Unreal foundations and need a cleaner way to manage game project assets, code, levels, exports, and documentation.

## Lesson Purpose

Students will organize a messy game development ZIP into a folder structure that separates project files, art, audio, code, levels, documents, exports, plugins, and prefabs. This prepares students to work like a small game team.

## Required Resources

- File 2 ZIP: https://drive.google.com/file/d/1X1vAIX0L7c5dvFrtmAZhQbz9XuRP7Kee/view?usp=sharing
- File 2 instructions: https://docs.google.com/document/d/1C2Po4zkeuFMmcJfTPx2OvCsmSl5mBu4GIZcFj7XMR1M/edit?usp=sharing

## Essential Understanding

A game team can work faster and avoid broken references when art, audio, code, levels, builds, documents, plugins, and prefabs are separated into predictable locations.

## Suggested 15-20 Slide Structure

1. Title and objective
2. The messy-game-project problem
3. Real development consequences: lost assets, broken builds, confusing versions
4. Game project folder overview
5. `01_Project_Files`
6. `02_Art`
7. Characters, Environments, and UI
8. `03_Audio`
9. `04_Code`
10. Scripts versus shaders
11. `05_Levels`
12. `06_Documents`
13. `07_Exports`
14. `08_Assets`
15. Filename clues and extension clues
16. Teacher sorting demo
17. Student workflow checklist
18. Evidence expectations
19. Common mistakes to avoid
20. Exit ticket and cleanup

## Slide-By-Slide Teaching Purpose

- Slides 1-3 should establish why organization matters in game development.
- Slides 4-14 should make the folder structure visible and concrete.
- Slides 15-16 should model how to reason from filenames and extensions.
- Slides 17-19 should support independent work and reduce common mistakes.
- Slide 20 should close with evidence, reflection, and cleanup.

## Must Include

- Clear folder-tree visual.
- At least six file examples from the ZIP.
- Before/after comparison.
- Evidence checklist.
- No raw upload directions; students submit Drive or Docs links through DCC Creative Studio.

## Tone

Practical, game-team focused, and teacher-led. This should feel like a real studio workflow, not a generic computer file lesson.
