# Video Game Development Folder Organization Assignment Sheet

## Objective

I can organize game development files into a professional project folder structure so art, audio, code, levels, documents, exports, plugins, and prefabs are easy to find.

## Assignment Resources

- File 2 ZIP: https://drive.google.com/file/d/1X1vAIX0L7c5dvFrtmAZhQbz9XuRP7Kee/view?usp=sharing
- File 2 instructions: https://docs.google.com/document/d/1C2Po4zkeuFMmcJfTPx2OvCsmSl5mBu4GIZcFj7XMR1M/edit?usp=sharing

## Skill Focus

- game project structure
- asset organization
- file type recognition
- development workflow
- evidence documentation

## Required Steps

1. Open the File 2 instructions.
2. Download File 2 and unzip it on your computer.
3. Create a new main game project folder using the class naming convention.
4. Build the required folder structure exactly.
5. Sort project files, art, audio, code, levels, documents, exports, and reusable assets into the correct folders.
6. Use file names and file extensions as clues when sorting.
7. Check that no files are left loose in the main project folder.
8. Capture a screenshot of the completed folder tree.
9. Create a short placement map listing at least twelve files and where you placed them.
10. Upload evidence to Google Drive or Google Docs and submit the share links in DCC Creative Studio.

## Sorting Hints

- `.uproject`, `.unity`, and `.blend` files belong in `01_Project_Files`.
- `.fbx`, `.obj`, and model-oriented files belong in `02_Art`.
- Character models belong in `02_Art/Characters`.
- Environment models and textures belong in `02_Art/Environments`.
- UI images such as buttons, menu backgrounds, and health bars belong in `02_Art/UI`.
- Music files belong in `03_Audio/Music`.
- Effects such as jumps, explosions, and sword clashes belong in `03_Audio/SoundFX`.
- `.cpp`, `.h`, `.cs`, and similar script files belong in `04_Code/Scripts`.
- Shader files such as `.usf`, `.shader`, `.hlsl`, and `.glsl` belong in `04_Code/Shaders`.
- `.umap` and scene files belong in `05_Levels`.
- Design documents, backstories, and flowcharts belong in `06_Documents`.
- `.exe` and `.apk` files belong in `07_Exports/Builds`.
- Plugin and prefab files belong in `08_Assets`.

## Naming Convention

Use this format when naming screenshots, written evidence, or submitted files: `LastName_FirstName_UE-Q2-FILEORG_Description`.

## Google Drive Link Evidence Requirements

- Google Drive link to a screenshot showing the completed game development folder tree.
- Google Drive or Docs link listing where at least twelve sample files were placed and why.
- Short reflection explaining two organization decisions that would help a game team work faster.

## What To Do If Stuck

- Look at the file extension first.
- Look for words in the file name such as `Hero`, `Enemy`, `Level`, `Shader`, `Plugin`, `Prefab`, `Build`, or `UI`.
- Ask whether the file is project setup, art, audio, code, level design, documentation, export, or reusable asset.

## 4-Point Rubric

- 4: Complete folder structure, all files sorted accurately, evidence is clear, and reflection explains thoughtful development workflow decisions.
- 3: Complete folder structure, most files sorted accurately, and all required evidence is submitted.
- 2: Folder structure or sorting is partially complete, or required evidence is missing detail.
- 1: Attempted but incomplete, hard to verify, or many files remain misplaced.
- 0: Not submitted.

## Extension Challenge

Create a one-page handoff guide explaining how another game developer should find art, code, levels, and builds in your organized folder.

## Reflection Prompt

Which two sorting decisions would save the most time during a real game development project, and why?
