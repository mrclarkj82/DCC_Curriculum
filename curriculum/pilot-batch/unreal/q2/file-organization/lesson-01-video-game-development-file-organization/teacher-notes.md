# Teacher Notes: Video Game Development File Organization

## Prep

- Confirm students can open the File 2 ZIP and instruction document.
- Decide where students should create the practice game project folder.
- Remind students that the website collects Drive or Docs links only. They should not upload raw folders to DCC Creative Studio.

## Lesson Flow

1. Open with a messy game project scenario and ask students what could go wrong.
2. Show the folder structure and explain why each top-level folder exists.
3. Model sorting three files out loud: one art asset, one code file, and one export/build.
4. Release students to build their folders and sort the full ZIP.
5. Circulate for correct folder naming, no loose files, and evidence quality.
6. Close with a reflection about which organization choices would help a game team work faster.

## Common Misconceptions

- Students may put builds in project files instead of exports.
- Students may mix UI graphics with environment textures.
- Students may treat plugins and prefabs as code instead of reusable assets.
- Students may leave files loose in the main folder after building subfolders.

## Evidence Check

Students should submit:

- Screenshot of the finished folder tree.
- Placement map for at least twelve files.
- Reflection naming two workflow decisions.

## Extension

Ask early finishers to write a short handoff note for another student joining the game team.
