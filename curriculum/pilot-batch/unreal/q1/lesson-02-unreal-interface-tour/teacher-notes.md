# Teacher Notes: Unreal Interface Tour

## Purpose of the Lesson

Students should be able to identify the main Unreal interface panels and explain how each one supports building and organizing a 3D world. This lesson is part of the pilot-ready Unreal Engine Studio content batch.

## Suggested 90-Minute Block Schedule

- 0-8 min: Bell ringer, attendance, and setup check
- 8-18 min: Teacher overview and vocabulary
- 18-35 min: Guided transcript-aligned demo
- 35-65 min: Student work time with teacher circulation
- 65-78 min: Evidence capture, naming, and submission support
- 78-87 min: Extension or cleanup time
- 87-90 min: Exit ticket and save reminder

## Teacher Mini-Lesson Talking Points

- Point out the Viewport as the view into the 3D world.
- Use the Toolbar to add a simple shape or switch modes briefly.
- Select objects from the Outliner and show multiple selection.
- Use the Details panel to inspect transform properties.
- Open the Content Drawer with Ctrl+Space and dock it as the Content Browser.
- Reset the editor layout using Window > Load Layout > Default Editor Layout.

## Likely Student Issues

- **Content Drawer disappears:** Ctrl+Space toggles it and Dock in Layout turns it into the Content Browser.
- **Student edits the wrong object:** Have them select objects through the Outliner and verify the Details panel name.
- **Layout becomes confusing:** Use Window > Load Layout > Default Editor Layout to reset the workspace.
- **Student thinks Content Drawer and Content Browser are different tools:** Clarify that they are two views of the same project asset area.

## Intervention Ideas

- Pair a stuck student with a student who can name the current tool without taking over the mouse.
- Ask students to point to the exact panel or gizmo before explaining the problem.
- Use the transcript timestamps as a rewind point rather than re-teaching the entire process.
- Keep a projected checklist visible during independent work time.

## Extension Ideas

Add two extra interface areas or shortcuts, such as World Settings, F10, Ctrl+Space, or Default Editor Layout.

## Vocabulary Emphasis

- **Viewport:** The main window that shows the 3D world and lets users select or manipulate objects.
- **Toolbar:** The row of tools used to add objects, switch modes, play, and access common actions.
- **Outliner:** A list of objects in the current level, useful for selecting, organizing, hiding, and grouping.
- **Details Panel:** The panel that shows editable properties for the selected object.
- **Content Drawer:** The asset browser that can open from the bottom of the editor, often with Ctrl+Space.
- **Content Browser:** The docked version of the Content Drawer that shows project folders and assets.

## What Evidence To Check Before Students Leave

- Annotated screenshot or form response identifying viewport, toolbar, outliner, details panel, and content drawer
