# Starter Master Material Variations Assignment Sheet

## Objective

I can describe why master materials help organize reusable material workflows.

## Skill Focus

- Master materials organize reusable material logic
- Neutral default values keep instances predictable
- Texture parameters allow instances to swap maps
- Static switch parameters can enable optional features
- Clear naming keeps materials and material instances readable

## Required Steps

1. Open or create the starter master material.
2. Check that default values are neutral and predictable.
3. Expose texture inputs for Base Color, Roughness, and Normal where appropriate.
4. Add or inspect a Metallic texture option.
5. Use or identify a Static Switch Parameter for optional metallic behavior.
6. Press Apply and Save.
7. Create at least two material instances from the master material.
8. Change parameters or texture slots to create controlled variations.
9. Apply the variations to objects in the level.
10. Capture screenshots that show both the setup and the visible results.

## Naming Convention

Use this format when naming screenshots or exported evidence: LastName_FirstName_UE-Q1-A10_Description.

## Screenshot/File Evidence Requirements

- Screenshot of the master material or teacher-provided master material parameters.
- Screenshot of a material instance with texture or value parameters enabled.
- Screenshot of at least two applied material variations in the level.
- Short reflection explaining one benefit of a master material.

## What To Do If Stuck

- Making the master material too specific to one surface.
- Leaving non-neutral default values that make every instance start with the wrong look.
- Forgetting to expose a texture as a parameter before trying to swap it in an instance.
- Using a metallic map on non-metal materials without a switch or default off state.
- Using inconsistent names that make parent materials and instances hard to identify.

## 4-Point Rubric

- 4: Complete, polished, on time, and clearly meets or extends the stated requirements.
- 3: Complete and meets the stated requirements.
- 2: Partially complete or missing required evidence.
- 1: Attempted but incomplete, unclear, or not functional.
- 0: Not submitted.

## Extension Challenge

Design a naming system for a small material library and label which instances would be used for floor, wall, prop, or metal objects.

## Reflection Prompt

Explain one way a master material saves time or reduces mistakes in a larger Unreal project.
