# Material Instance Variation Set Assignment Sheet

## Objective

I can explain core material concepts and create an editable material instance.

## Skill Focus

- Material instances inherit from parent materials
- Parameters expose safe values for editing
- Scalar and vector parameters control different kinds of material data
- Material instances update faster than rebuilding separate materials
- Apply and Save make new parent-material parameters available to instances

## Required Steps

1. Complete Quiz 2 or record teacher checkoff.
2. Open the parent material from the previous texture-map workflow.
3. Convert a scalar value, such as size or roughness strength, into a parameter.
4. Convert a color value into a vector parameter.
5. Press Apply and Save so the instance can access the new parameters.
6. Create a material instance using the MI_ naming convention.
7. Apply the material instance to an object or floor surface.
8. Enable and adjust at least two exposed parameters.
9. Create a second variation with a different readable look.
10. Capture evidence showing the parent material and the two variations.

## Naming Convention

Use this format when naming screenshots or exported evidence: LastName_FirstName_UE-Q1-A09_Description.

## Screenshot/File Evidence Requirements

- Quiz 2 completion record or teacher checkoff.
- Screenshot of the parent material showing at least two exposed parameters.
- Screenshot of at least two material instance variations applied in the level.
- Short reflection explaining how a material instance is different from a separate new material.

## What To Do If Stuck

- Creating a brand-new material instead of a material instance.
- Forgetting to press Apply after converting a value to a parameter.
- Editing the parent material when the task asks for instance variations.
- Not enabling the checkbox beside a material instance parameter.
- Submitting variations that look identical in the evidence screenshots.

## 4-Point Rubric

- 4: Complete, polished, on time, and clearly meets or extends the stated requirements.
- 3: Complete and meets the stated requirements.
- 2: Partially complete or missing required evidence.
- 1: Attempted but incomplete, unclear, or not functional.
- 0: Not submitted.

## Extension Challenge

Create three named variations that could belong to one environment set, such as clean, worn, and stylized.

## Reflection Prompt

Explain how a material instance changes workflow speed compared with creating a brand-new material every time.
