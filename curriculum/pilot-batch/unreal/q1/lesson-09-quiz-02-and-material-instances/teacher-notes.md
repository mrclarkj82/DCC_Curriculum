# Quiz 2 and Material Instances Teacher Notes

## Purpose of the Lesson

Students turn previous material knowledge into a more efficient production workflow. The lesson links Quiz 2 review to practical material instance variation so students see why parameters matter.

## Suggested 90-Minute Block Schedule

- 0-7 minutes: Login, project setup, and bell ringer.
- 7-17 minutes: Teacher slides and success criteria.
- 17-35 minutes: Video/demo segment.
- 35-75 minutes: Unreal build task with teacher check-ins.
- 75-85 minutes: Submission evidence and file naming.
- 85-90 minutes: Exit ticket, save, and cleanup.

## Teacher Mini-Lesson Talking Points

- Complete or review Quiz 2 before opening the new workflow.
- Find the parent material in the Content Drawer.
- Right-click the parent material and create a material instance.
- Use MI_ naming for the instance and open it.
- Show that no parameters appear until values are exposed in the parent material.
- Convert a scalar value to a parameter and name it clearly.
- Press Apply and Save in the parent material.
- Apply the material instance to a scene object.
- Enable the parameter checkbox in the instance and adjust the value.
- Repeat with a color/vector parameter and compare variations.

## Likely Student Issues

- Creating a brand-new material instead of a material instance.
- Forgetting to press Apply after converting a value to a parameter.
- Editing the parent material when the task asks for instance variations.
- Not enabling the checkbox beside a material instance parameter.
- Submitting variations that look identical in the evidence screenshots.

## Intervention Ideas

- Ask students to point to the exact setting, node, object, or screenshot that proves the skill.
- Pair students for a 2-minute evidence check before submission.
- For students who finish early, move them into the extension challenge rather than adding unrelated features.

## Extension Ideas

Create three named variations that could belong to one environment set, such as clean, worn, and stylized.

## Vocabulary Emphasis

- Material instance: A child version of a material that can expose editable settings without rebuilding the whole material graph.
- Parent material: The original material that controls the structure and available parameters for its instances.
- Parameter: A material value exposed so it can be edited from a material instance.
- Scalar parameter: A single-number parameter, often used for values such as size, roughness strength, or normal flatness.
- Vector parameter: A multi-value parameter often used for color tint or RGB values.
- Texture parameter: A texture slot exposed so a material instance can swap a texture map.
- Override: An enabled material instance value that replaces the parent material default.
- Real-time update: A change that appears immediately in the viewport without recompiling the full material.
- Reset to default: Returning an overridden material instance value back to the parent material value.
- Apply: The material editor action that updates the parent material so new parameters appear in instances.

## What Evidence To Check Before Students Leave

- Quiz 2 completion record or teacher checkoff.
- Screenshot of the parent material showing at least two exposed parameters.
- Screenshot of at least two material instance variations applied in the level.
- Short reflection explaining how a material instance is different from a separate new material.
