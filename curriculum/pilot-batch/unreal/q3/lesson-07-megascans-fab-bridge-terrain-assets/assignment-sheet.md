# Terrain Asset Import and Placement Assignment Sheet

## Objective

I can import and place terrain assets responsibly while documenting how assets support the castle environment.

## Skill Focus

- asset import
- asset placement
- scene consistency
- screen recording
- production log

## Required Steps

1. Import or choose terrain assets appropriate for the castle environment.
2. Place assets to support the route, silhouette, or story.
3. Adjust scale, rotation, and placement so assets feel intentional.
4. Record a clip showing one asset before and after placement.
5. Update the production log with why the asset was chosen.

## File / Project Naming Convention

Use this format when naming project folders, screenshots, recordings, production logs, and written evidence: LastName_FirstName_UE-Q3-A07_Description.

## Evidence Requirements

- Screenshot of imported terrain assets placed in the scene.
- Daily documentary clip showing an asset placement decision.
- Production log entry for today.

## Google Drive Link Submission Requirements

- Upload screenshots, screen recordings, camera clips, production log notes, or documentary draft evidence to Google Drive or YouTube according to teacher directions.
- Set link sharing so the teacher can view the evidence.
- Paste the evidence link into DCC Creative Studio.
- Do not upload raw media files directly to the website.
- If the evidence is a folder, make sure every required file inside the folder is viewable.

## What To Do If Stuck

- Re-read the learning target and identify the one visible skill you need to prove today.
- Capture a short before clip before making large changes.
- Ask whether the issue is project organization, camera view, scale, terrain, materials, assets, lighting, or evidence capture.
- Use screenshots plus production notes if screen recording fails.
- Keep working on the castle project while documenting the decision you can explain.

## Four-Point Rubric

- 4: Complete, polished terrain asset import and placement; daily documentary evidence is clear, organized, and strongly explains process decisions.
- 3: Complete terrain asset import and placement; daily documentary evidence and production log are present.
- 2: Partially complete terrain asset import and placement; documentary evidence or production log is incomplete or unclear.
- 1: Attempted but incomplete; little process evidence is visible.
- 0: Not submitted.

## Extension Challenge

Make a mini asset list explaining which assets are decorative, navigational, or story-based.

## Studio Challenge

Add one optional behind-the-scenes clip or caption that could help the final documentary explain today's build decision.

## Reflection Prompt

How did one asset make the scene feel more complete?
