# Third-Person Walkthrough Setup Assignment Sheet

## Objective

I can set up a playable third-person Unreal project and document why player scale matters for a castle walkthrough.

## Skill Focus

- third-person setup
- player scale
- testing
- screen recording
- production log

## Required Steps

1. Set up or confirm a third-person project/template configuration.
2. Place or test the player character in the castle environment workspace.
3. Check player scale against temporary terrain or blockout objects.
4. Record a short walkthrough clip from the player perspective.
5. Update the production log with what the player test revealed.

## File / Project Naming Convention

Use this format when naming project folders, screenshots, recordings, production logs, and written evidence: LastName_FirstName_UE-Q3-A03_Description.

## Evidence Requirements

- Screenshot of the playable character in the project.
- Short note explaining why player scale matters.
- Daily screen recording of a player-view test.
- Production log entry for today.

## Google Drive Link Submission Requirements

- Upload screenshots, screen recordings, camera clips, production log notes, or documentary draft evidence to Google Drive or YouTube according to teacher directions.
- Set link sharing so the teacher can view the evidence.
- Paste the evidence link into DCC Creative Studio.
- Do not upload raw media files directly to the website.
- If the evidence is a folder, make sure every required file inside the folder is viewable.

## What To Do If Stuck

- Re-read the learning target and identify the one visible skill you need to prove today.
- Capture a short before clip before making large changes.
- Ask whether the issue is project organization, camera view, scale, terrain, materials, assets, lighting, or evidence capture.
- Use screenshots plus production notes if screen recording fails.
- Keep working on the castle project while documenting the decision you can explain.

## Four-Point Rubric

- 4: Complete, polished third-person walkthrough setup; daily documentary evidence is clear, organized, and strongly explains process decisions.
- 3: Complete third-person walkthrough setup; daily documentary evidence and production log are present.
- 2: Partially complete third-person walkthrough setup; documentary evidence or production log is incomplete or unclear.
- 1: Attempted but incomplete; little process evidence is visible.
- 0: Not submitted.

## Extension Challenge

Mark one intended player route through the future castle environment.

## Studio Challenge

Add one optional behind-the-scenes clip or caption that could help the final documentary explain today's build decision.

## Reflection Prompt

What changed when you evaluated the scene from player view instead of editor view?
