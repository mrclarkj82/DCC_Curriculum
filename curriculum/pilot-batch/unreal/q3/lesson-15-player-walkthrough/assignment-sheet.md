# Castle Player Walkthrough Recording Assignment Sheet

## Objective

I can record a player walkthrough of the castle environment and evaluate what the walkthrough communicates about the build.

## Skill Focus

- player walkthrough
- recording
- route readability
- critique
- production log

## Required Steps

1. Plan a short player route through or around the castle.
2. Record a screen capture walkthrough from player view.
3. Review the recording for pacing, readability, and visual issues.
4. Identify at least one final improvement based on the recording.
5. Update the production log with what the walkthrough revealed.

## File / Project Naming Convention

Use this format when naming project folders, screenshots, recordings, production logs, and written evidence: LastName_FirstName_UE-Q3-A15_Description.

## Evidence Requirements

- Google Drive or YouTube link to a player walkthrough recording.
- Screenshot or note identifying one final improvement.
- Production log entry for today.

## Google Drive Link Submission Requirements

- Upload screenshots, screen recordings, camera clips, production log notes, or documentary draft evidence to Google Drive or YouTube according to teacher directions.
- Set link sharing so the teacher can view the evidence.
- Paste the evidence link into DCC Creative Studio.
- Do not upload raw media files directly to the website.
- If the evidence is a folder, make sure every required file inside the folder is viewable.

## What To Do If Stuck

- Re-read the learning target and identify the one visible skill you need to prove today.
- Capture a short before clip before making large changes.
- Ask whether the issue is project organization, camera view, scale, terrain, materials, assets, lighting, or evidence capture.
- Use screenshots plus production notes if screen recording fails.
- Keep working on the castle project while documenting the decision you can explain.

## Four-Point Rubric

- 4: Complete, polished castle player walkthrough recording; daily documentary evidence is clear, organized, and strongly explains process decisions.
- 3: Complete castle player walkthrough recording; daily documentary evidence and production log are present.
- 2: Partially complete castle player walkthrough recording; documentary evidence or production log is incomplete or unclear.
- 1: Attempted but incomplete; little process evidence is visible.
- 0: Not submitted.

## Extension Challenge

Record a second walkthrough after making one improvement and compare the two clips.

## Studio Challenge

Add one optional behind-the-scenes clip or caption that could help the final documentary explain today's build decision.

## Reflection Prompt

What did the walkthrough reveal that screenshots did not?
